Project Background
This project is part of a Hand Gesture Recognition System that uses MediaPipe Hand Landmarks instead of raw images.
Rather than training a machine learning model directly on images, the project extracts the 21 hand landmark coordinates (x, y, z) detected by MediaPipe and converts them into numerical features. These numerical features are then used as input for machine learning models such as Random Forest, SVM, KNN, Logistic Regression, XGBoost, etc.

7. Cross Validation Strategy
Originally
StratifiedShuffleSplit
was used.
Later it was found that this is not suitable for person-independent evaluation.
Reason:
The same person's samples may appear in both training and testing.
This introduces subject leakage.
________________


8. Desired Cross Validation
The desired approach is
GroupKFold
using
groups = person_id
Advantages
* no person leakage
* person-independent testing
* stronger evaluation
* academically more acceptable
Each fold keeps all samples from one person entirely in either training or testing.
________________

13. Problems Encountered
Problem 1
Using
train_idx[:n_train]
created biased training subsets because the indices were not randomized.
Solution
Shuffle the training indices before applying the training ratio.
________________


Problem 2
XGBoost requires class labels to be consecutive integers starting from zero.
Our joint labels are
15
216
327
4108
...
Therefore XGBoost raised
Invalid classes inferred from unique values of y
________________


Problem 3
Encoding labels separately inside each fold caused
ValueError


y contains previously unseen labels
because GroupKFold creates person-independent splits.
Some gesture-person combinations exist only in the test fold.
They never appear during training.
________________


14. Current Status
The pipeline works correctly for
Decision Tree
Random Forest
KNN
Naive Bayes
SVM
Logistic Regression
The remaining issue concerns XGBoost with GroupKFold and joint gesture-person labels.
The problem arises because XGBoost expects every class present in the test set to also exist in the training set.
However, person-independent GroupKFold intentionally creates unseen joint classes in some folds.
